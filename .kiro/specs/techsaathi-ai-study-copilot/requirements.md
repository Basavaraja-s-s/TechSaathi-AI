# Requirements Document

## Introduction

TechSaathi AI – Smart Study Copilot is an AI-powered web application designed to assist students with their learning activities. The system provides AI tutoring, code debugging assistance, exam question generation, study plan creation, and document-based question answering through a unified chat interface. The application uses Groq's LLaMA 3 model for AI capabilities, AWS S3 for document storage, and a modern web interface built with FastAPI, HTML, CSS, and vanilla JavaScript.

## Glossary

- **TechSaathi_System**: The complete AI study copilot web application
- **Chat_Workspace**: The main interface where users interact with the AI assistant
- **AI_Service**: The backend service that processes AI requests using Groq API
- **PDF_Service**: The service that extracts text from uploaded PDF documents
- **S3_Service**: The service that manages document storage in AWS S3
- **Chat_Mode**: The operational mode determining how the AI processes requests (chat, code, exam, study_plan, document)
- **Document_Context**: The extracted text content from uploaded PDF files used for answering questions
- **Study_Plan**: A structured 7-day learning schedule generated by the AI
- **Exam_Questions**: A set of 5 questions with answers generated for assessment purposes
- **User_Message**: Text input submitted by the student user
- **AI_Response**: Text output generated by the AI assistant
- **Theme**: The visual appearance mode (dark or light) of the user interface
- **Dashboard**: The statistics page showing usage metrics
- **Message_Bubble**: The visual container displaying a single chat message
- **Typing_Indicator**: The animated visual feedback showing the AI is processing

## Requirements

### Requirement 1: AI Chat Interaction

**User Story:** As a student, I want to send messages to the AI assistant and receive intelligent responses, so that I can get help with my studies.

#### Acceptance Criteria

1. WHEN a User_Message is submitted, THE Chat_Workspace SHALL display the message in a Message_Bubble
2. WHEN a User_Message is received by the backend, THE AI_Service SHALL send the message to Groq API with model llama3-8b-8192
3. WHILE the AI_Service is processing, THE Chat_Workspace SHALL display a Typing_Indicator
4. WHEN the AI_Response is received, THE Chat_Workspace SHALL display the response in a Message_Bubble
5. THE Chat_Workspace SHALL support markdown formatting in AI_Response messages
6. THE Chat_Workspace SHALL support code block formatting with syntax highlighting in AI_Response messages
7. WHEN the user presses Enter key, THE Chat_Workspace SHALL submit the User_Message
8. WHEN the user presses Shift+Enter, THE Chat_Workspace SHALL insert a new line in the input field
9. THE Chat_Workspace SHALL automatically scroll to show the latest message

### Requirement 2: Chat Mode Routing

**User Story:** As a student, I want to select different modes for specialized assistance, so that I can get targeted help for specific tasks.

#### Acceptance Criteria

1. THE TechSaathi_System SHALL support five Chat_Mode options: chat, code, exam, study_plan, and document
2. WHEN Chat_Mode is "code", THE AI_Service SHALL analyze code for bugs and suggest improvements
3. WHEN Chat_Mode is "exam", THE AI_Service SHALL generate exactly 5 Exam_Questions with answers
4. WHEN Chat_Mode is "study_plan", THE AI_Service SHALL generate a 7-day Study_Plan
5. WHEN Chat_Mode is "document", THE AI_Service SHALL answer questions using Document_Context from uploaded PDFs
6. WHEN Chat_Mode is "chat", THE AI_Service SHALL provide general tutoring assistance
7. THE Chat_Workspace SHALL display the current Chat_Mode in the top navbar
8. WHEN the user changes Chat_Mode, THE Chat_Workspace SHALL update the mode selector display

### Requirement 3: PDF Document Upload and Processing

**User Story:** As a student, I want to upload PDF documents, so that I can ask questions about their content.

#### Acceptance Criteria

1. WHEN a PDF file is selected for upload, THE TechSaathi_System SHALL accept the file through the upload endpoint
2. WHEN a PDF file is received, THE PDF_Service SHALL extract text content from all pages
3. WHEN text extraction completes, THE S3_Service SHALL upload the PDF file to AWS S3
4. WHEN the PDF is uploaded to S3, THE AI_Service SHALL generate a summary of the document content
5. WHEN upload processing completes, THE TechSaathi_System SHALL return the document metadata including filename, S3 URL, and summary
6. IF the PDF file is corrupted or unreadable, THEN THE PDF_Service SHALL return an error message
7. THE Chat_Workspace SHALL display an upload button in the left sidebar
8. WHEN a document is successfully uploaded, THE Chat_Workspace SHALL add it to the documents list in the sidebar

### Requirement 4: Document Management and Retrieval

**User Story:** As a student, I want to view my uploaded documents and select them for questions, so that I can reference specific materials.

#### Acceptance Criteria

1. THE TechSaathi_System SHALL provide an endpoint that returns all uploaded documents from S3
2. WHEN the Chat_Workspace loads, THE TechSaathi_System SHALL fetch and display the list of uploaded documents
3. THE Chat_Workspace SHALL display each document name in the left sidebar
4. WHEN a user selects a document from the sidebar, THE Chat_Workspace SHALL load the Document_Context for that file
5. WHEN Chat_Mode is "document" and a document is selected, THE AI_Service SHALL use the Document_Context to answer questions
6. THE Chat_Workspace SHALL visually indicate which document is currently selected

### Requirement 5: Backend API Endpoints

**User Story:** As a developer, I want well-defined API endpoints, so that the frontend can communicate with the backend services.

#### Acceptance Criteria

1. THE TechSaathi_System SHALL provide a GET endpoint at "/" that serves the main Chat_Workspace page
2. THE TechSaathi_System SHALL provide a POST endpoint at "/chat" that accepts JSON with message and mode fields
3. WHEN the "/chat" endpoint receives a request, THE TechSaathi_System SHALL return an AI_Response in JSON format
4. THE TechSaathi_System SHALL provide a POST endpoint at "/upload-pdf" that accepts multipart form data with a PDF file
5. WHEN the "/upload-pdf" endpoint receives a file, THE TechSaathi_System SHALL return document metadata in JSON format
6. THE TechSaathi_System SHALL provide a GET endpoint at "/documents" that returns a JSON array of document metadata
7. IF any endpoint receives invalid data, THEN THE TechSaathi_System SHALL return an appropriate HTTP error status code with an error message

### Requirement 6: User Interface Layout and Navigation

**User Story:** As a student, I want an intuitive and modern interface, so that I can easily navigate and use the application.

#### Acceptance Criteria

1. THE Chat_Workspace SHALL display a top navbar containing the app name, mode selector, and theme toggle
2. THE Chat_Workspace SHALL display a left sidebar containing the documents list, upload button, and new chat button
3. THE Chat_Workspace SHALL display a main chat area containing Message_Bubble elements
4. THE Chat_Workspace SHALL display a bottom input area containing a text box, send button, and attach file button
5. WHEN the user clicks the new chat button, THE Chat_Workspace SHALL clear all messages and start a fresh conversation
6. THE Chat_Workspace SHALL use rounded corners for Message_Bubble elements
7. THE Chat_Workspace SHALL use smooth CSS transitions for UI state changes
8. THE Chat_Workspace SHALL be responsive and adapt to different screen sizes

### Requirement 7: Theme Customization

**User Story:** As a student, I want to switch between dark and light themes, so that I can use the app comfortably in different lighting conditions.

#### Acceptance Criteria

1. THE TechSaathi_System SHALL support two Theme options: dark and light
2. THE Chat_Workspace SHALL implement Theme styling using CSS variables
3. WHEN the user clicks the theme toggle, THE Chat_Workspace SHALL switch between dark and light Theme
4. WHEN Theme changes, THE Chat_Workspace SHALL apply smooth transitions to color changes
5. THE Chat_Workspace SHALL persist the selected Theme in browser local storage
6. WHEN the Chat_Workspace loads, THE Chat_Workspace SHALL apply the previously selected Theme from local storage

### Requirement 8: Message Interaction Features

**User Story:** As a student, I want to interact with chat messages, so that I can copy useful information and review content easily.

#### Acceptance Criteria

1. THE Chat_Workspace SHALL display a copy button on each AI_Response Message_Bubble
2. WHEN the user clicks the copy button, THE Chat_Workspace SHALL copy the message text to the system clipboard
3. WHEN text is copied, THE Chat_Workspace SHALL display a visual confirmation indicator
4. THE Chat_Workspace SHALL distinguish between user and AI messages with different Message_Bubble styles
5. THE Chat_Workspace SHALL display timestamps or message ordering indicators

### Requirement 9: Dashboard Statistics

**User Story:** As a student, I want to see my usage statistics, so that I can track my learning activity.

#### Acceptance Criteria

1. THE TechSaathi_System SHALL provide a Dashboard page accessible via navigation
2. THE Dashboard SHALL display the total number of uploaded documents
3. THE Dashboard SHALL display the total number of chat interactions
4. THE Dashboard SHALL display the total number of Study_Plan items generated
5. WHEN the Dashboard loads, THE TechSaathi_System SHALL fetch and display current statistics
6. THE Dashboard SHALL use the same Theme styling as the Chat_Workspace

### Requirement 10: Application Configuration and Deployment

**User Story:** As a developer, I want clear configuration and deployment instructions, so that I can run the application locally.

#### Acceptance Criteria

1. THE TechSaathi_System SHALL provide a requirements.txt file listing all Python dependencies
2. THE TechSaathi_System SHALL provide a .env.example file showing required environment variables
3. THE TechSaathi_System SHALL require environment variables for Groq API key and AWS S3 credentials
4. THE TechSaathi_System SHALL start when running "uvicorn main:app --reload"
5. WHEN the application starts, THE TechSaathi_System SHALL be accessible at http://localhost:8000
6. THE TechSaathi_System SHALL organize code into services directory for AI, PDF, and S3 functionality
7. THE TechSaathi_System SHALL organize frontend files into templates and static directories

### Requirement 11: Code Debugging Assistance

**User Story:** As a student learning to code, I want the AI to help me debug code, so that I can understand and fix errors.

#### Acceptance Criteria

1. WHEN Chat_Mode is "code" and a User_Message contains code, THE AI_Service SHALL analyze the code for syntax errors
2. WHEN Chat_Mode is "code" and a User_Message contains code, THE AI_Service SHALL identify logical bugs
3. WHEN Chat_Mode is "code" and a User_Message contains code, THE AI_Service SHALL suggest specific improvements
4. WHEN Chat_Mode is "code" and a User_Message contains code, THE AI_Service SHALL explain what the code does
5. THE AI_Response SHALL format code suggestions using code blocks with syntax highlighting

### Requirement 12: Exam Question Generation

**User Story:** As a student preparing for exams, I want the AI to generate practice questions, so that I can test my knowledge.

#### Acceptance Criteria

1. WHEN Chat_Mode is "exam" and a User_Message specifies a topic, THE AI_Service SHALL generate exactly 5 Exam_Questions
2. FOR EACH generated question, THE AI_Service SHALL provide a complete answer
3. THE AI_Response SHALL format Exam_Questions in a numbered list
4. THE AI_Response SHALL clearly separate questions from answers
5. THE Exam_Questions SHALL be relevant to the topic specified in the User_Message

### Requirement 13: Study Plan Generation

**User Story:** As a student, I want the AI to create a study plan, so that I can organize my learning schedule.

#### Acceptance Criteria

1. WHEN Chat_Mode is "study_plan" and a User_Message specifies a subject, THE AI_Service SHALL generate a 7-day Study_Plan
2. THE Study_Plan SHALL include specific topics or activities for each day
3. THE Study_Plan SHALL be structured with clear day-by-day breakdown
4. THE AI_Response SHALL format the Study_Plan in an organized, readable layout
5. THE Study_Plan SHALL be realistic and achievable within the 7-day timeframe

### Requirement 14: Error Handling and User Feedback

**User Story:** As a student, I want clear error messages when something goes wrong, so that I understand what happened and what to do next.

#### Acceptance Criteria

1. IF the Groq API request fails, THEN THE AI_Service SHALL return a user-friendly error message
2. IF the S3_Service cannot connect to AWS, THEN THE TechSaathi_System SHALL return an error message indicating storage issues
3. IF the PDF_Service cannot extract text from a file, THEN THE TechSaathi_System SHALL return an error message explaining the issue
4. IF the user submits an empty User_Message, THEN THE Chat_Workspace SHALL prevent submission and display a validation message
5. IF a network request fails, THEN THE Chat_Workspace SHALL display an error notification to the user
6. WHEN an error occurs, THE Chat_Workspace SHALL remove the Typing_Indicator

### Requirement 15: PDF Text Extraction and Parsing

**User Story:** As a developer, I want reliable PDF text extraction, so that document-based questions can be answered accurately.

#### Acceptance Criteria

1. WHEN a PDF file is uploaded, THE PDF_Service SHALL extract text from all pages using pypdf
2. THE PDF_Service SHALL preserve text structure and formatting where possible
3. THE PDF_Service SHALL handle multi-page PDF documents
4. IF a PDF page contains no extractable text, THEN THE PDF_Service SHALL skip that page and continue processing
5. THE PDF_Service SHALL return the complete extracted text as Document_Context
6. FOR ALL valid PDF documents, THE PDF_Service SHALL successfully extract text or return a descriptive error
